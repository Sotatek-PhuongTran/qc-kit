## Ghi chú cột

> Mô tả ngắn cho người đọc. **Giá trị/format chi tiết từng cột** (điều kiện inject, format ô, luật parse điểm/verdict): xem `.claude/skills/qc-dashboard-sync/references/contracts.md` §2 (schema + vị trí cột) và §5 (per column).

### Cột cơ bản (luôn có)

- **{{ID_LABEL}}:** định danh canonical của feature/UC theo site-map. Tên cột adapt theo cách dự án gọi (`Use Case ID`, `Feature ID`, `Story ID`, ...).
- **Folder ID:** tên/ID folder thực tế trên disk, dùng để nối dòng ↔ folder khi scan. Chỉ điền khi KHÁC cột định danh; để trống nghĩa là trùng.
- **Tên feature/UC:** tên người-đọc-hiểu (site-map cung cấp; bottom-up để trống chờ user / site-map Mode 3 cập nhật).
- **In scope?:** `Yes` / `No` / `Need confirm`. `qc-dashboard-sync` KHÔNG tự đổi giá trị này; user sửa thủ công được bất cứ lúc nào.
- **Tài liệu nguồn:** tài liệu đầu vào tìm thấy trên disk cho UC — đặc tả (`Specs`) + wireframe (`WF`) kèm version cao nhất. Version các sản phẩm QC (audited, scenario, TC) hiển thị ở các cột trạng thái bên phải, không ở cột này.
- **Review stt:** kết quả review use case — verdict + điểm từ file audited version cao nhất (sản phẩm của `qc-uc-read`).
- **Scenario stt:** version scenario cao nhất trên disk (sản phẩm của `qc-func-scenario-design`).
- **TC stt:** version test case cao nhất trên disk (sản phẩm của `qc-func-tc-design`; ghi chú riêng khi mới có bản md, chưa có xlsx).

> Cột trạng thái đang có skill chạy dở → hiển thị nguyên văn dòng `status:` của checkpoint; chưa có sản phẩm → để TRỐNG (contracts §4/§5).

### Cột optional (auto-inject bởi `qc-dashboard-sync`; đã inject thì không tự xóa)

- **Site** (khi dự án có >1 site/portal): portal mà feature/UC thuộc về (Admin, Vendor, App...).
- **Module** (khi dự án có >1 module): module / nhóm chức năng.
- **API stt** (khi dự án có nhánh API): trạng thái thiết kế nhánh API gộp audit + scenario + TC.
- **Automation stt** (khi project automation tồn tại): số file test script (`.spec.ts`) của UC, tổng cả 2 nhánh (UI + API/MIX).
- **Execute stt** (khi project automation tồn tại): kết quả chạy gần nhất, copy từ file tổng hợp của `qc-auto-run`; giữ nguyên giá trị cũ khi lần chạy gần nhất không có UC này.
