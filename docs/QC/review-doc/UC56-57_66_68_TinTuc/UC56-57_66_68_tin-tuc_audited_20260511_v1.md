## Section 6 — Functional Logic & Workflow Decomposition

### 6.1 Chức năng: Xem danh sách bài viết

**MAIN FLOW (Happy Path):**
1. Người dùng chọn "Tin tức" từ Sidebar
2. Hệ thống mở màn hình Tin tức, Header hiển thị "Tin tức"
3. Hệ thống gọi API danh sách Tab, active "Tất cả"
4. Hệ thống gọi API danh sách bài viết theo Tab + ngôn ngữ hiện tại
5. Hiển thị Skeleton Loading cho 2 section
6. Sau khi tải: Tin nổi bật (carousel ngang) + Tin tức mới nhất (list dọc, mới nhất lên đầu)
7. Cuộn xuống cuối → Lazy load 20 bài tiếp (CMR-04)

**ALTERNATIVE FLOWS:**
- [Alt-1] Đổi tab → Tải lại danh sách theo category mới, giữ nguyên bộ lọc ngày (nếu có)
- [Alt-2] Pull to Refresh → Gọi lại API, làm mới cả 2 section (CMR-13)
- [Alt-3] Đổi ngôn ngữ → Lọc lại bài viết theo ngôn ngữ mới
- [Alt-4] Không có tin nổi bật → Ẩn section "Tin nổi bật", kéo "Tin tức mới nhất" lên

**EXCEPTION & ERROR FLOWS:**
- [Err-1] Lỗi mạng → "Không thể kết nối. Vui lòng kiểm tra mạng và thử lại." + nút "Thử lại" (CMR-07)
- [Err-2] API 500 → "Hệ thống đang bận. Vui lòng thử lại sau." (CMR-07)
- [Err-3] Timeout → "Yêu cầu đã hết thời gian chờ. Vui lòng thử lại." (CMR-07, CMR-16)
- [Err-4] API trả danh sách rỗng → "Không có dữ liệu." (CMR-14)

**BUSINESS RULES:**
- BR-01: Bài viết trong "Tin nổi bật" KHÔNG xuất hiện lại trong "Tin tức mới nhất" (exclusion rule)
- BR-02: Sắp xếp mặc định "Tin tức mới nhất" theo Latest (mới nhất lên đầu)
- BR-03: Bộ lọc ngày đăng áp dụng xuyên suốt tất cả Tab (không reset khi đổi tab)
- BR-04: Lọc bài viết theo ngôn ngữ hiển thị ứng dụng (không dịch nội dung)

**UI/UX FEEDBACK:**
- Loading state: Skeleton Loading (first-load), spinner (lazy load) — theo CMR-07
- Empty state: "Không có dữ liệu." ở giữa vùng nội dung (CMR-14)

**Gaps phát hiện:**
- ⚠️ Bài viết đánh dấu "Tin nổi bật" do flag từ API hay CMS? Cụ thể field nào? (chỉ cần biết để test hiển thị phía client)
- ⚠️ Khi đổi tab, Skeleton Loading có hiển thị lại không hay chỉ spinner? (CMR-07 nói first-load = full screen, nhưng đổi tab có tính first-load không?)

### 6.2 Chức năng: Tìm kiếm & Lọc

**MAIN FLOW — Tìm kiếm (Happy Path):**
1. Tap icon kính lúp → Thanh search mở rộng dưới header
2. Nhập từ khóa → Debounce 3s → Tìm kiếm gần đúng theo tiêu đề trong Tab active
3. Kết quả hiển thị tự động (CMR-01)

**MAIN FLOW — Lọc (Happy Path):**
